## CMS review findings

The main CMS problem is not a security bug. It is a truthfulness problem:

- The Sanity editor currently exposes content that does not control the live website.
- `featuredWorkSection.projects` exists in the editor, but the homepage does not read it.
- `masonryGallery` exists in the editor, but the gallery page does not read it.
- `workProject` exists mainly to support the unused `projects` field above.

### What the website actually reads

- Home featured images: `photo` documents with `featured == true`
- Home parallax strips: `featuredWorkSection.parallaxStrips`
- Gallery page: all `photo` documents grouped by category
- About page: `about`
- Shop page: `shopItem`

### Why this matters

Someone editing content in Sanity could reasonably think:

- changing "Featured Work > Projects" will change the homepage
- changing "Gallery" singleton will change the gallery page

Right now, neither is true.

## Result

The CMS is now more honest:

- Editors will no longer see controls for homepage projects or gallery curation that do nothing on the live site.
- The Sanity Studio now better matches the real data flow of the website.
